## ID-ключ и модификатор `readonly`

Иногда каждому объекту нужен **уникальный идентификатор (ID)**.

Например:
- первый клиент → `0`
- второй → `1`
- третий → `2`

Для этого используют **статическое поле** и `readonly`.

---

## Статический счётчик ID

```csharp
private static int nextId = 0;
```

Это общее поле для всего класса.

Хранит номер следующего ID.

---

## Поле `readonly`

```csharp
private readonly int _id;
```

`readonly` означает:

- значение можно присвоить **только один раз**:
  - при объявлении,
  - или в конструкторе.

После этого изменить нельзя:

```csharp
_id = 5; // Ошибка (если не в конструкторе)
```

---

## Присвоение ID в конструкторе

```csharp
_id = nextId++;
```

Как работает:

1. текущий `nextId` записывается в `_id`
2. потом `nextId` увеличивается на 1

---

Пример:

Создаём:

```csharp
Customer c1 = new Customer(...);
Customer c2 = new Customer(...);
Customer c3 = new Customer(...);
```

Получим:

```text
c1._id = 0
c2._id = 1
c3._id = 2
```

---

## Вывод ID

В статическом методе:

```csharp
Console.WriteLine($"Id: {customer._id}");
```

Результат:

```text
Id: 0
```

---

## Зачем `readonly`?

Чтобы ID нельзя было случайно изменить:

```csharp
customer._id = 100; // нельзя
```

Это защищает уникальность объекта.

---

## Идея

```text
static nextId -> общий счётчик
readonly _id -> постоянный ID объекта
```
