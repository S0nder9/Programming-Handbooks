## Определение логики установки свойства (`set`)

В `set` можно добавить **свою логику проверки** перед изменением значения свойства.

Пример:

```csharp
public string Model
{
    get => _model;

    set
    {
        if (string.IsNullOrEmpty(value))
        {
            Console.WriteLine("280");
            _model = "";
        }
        else
        {
            _model = value;
        }
    }
}
```

---

## Что делает `string.IsNullOrEmpty(value)`?

Проверяет:
- `null`
- пустую строку `""`

Если пользователь ничего не ввёл:

```text
Enter
```

то условие будет `true`.

---

## Логика работы

Если строка пустая:

```csharp
_model = "";
Console.WriteLine("280");
```

Если строка не пустая:

```csharp
_model = value;
```

---

## Использование

```csharp
Auto audi = new Auto("A3");

audi.Model = Console.ReadLine();

Console.WriteLine(audi.Model);
```

### Пример 1: ввод `"BMW"`

Пользователь ввёл:

```text
BMW
```

Результат:

```text
BMW
```

---

### Пример 2: пустой ввод

Пользователь нажал Enter:

```text

```

Результат:

```text
280
```

Потому что сработала проверка в `set`.

---

## Зачем это нужно?

Чтобы **контролировать данные**, которые записываются в объект.

Это часть **инкапсуляции**:
- пользователь меняет свойство,
- класс сам решает, разрешить изменение или нет.
