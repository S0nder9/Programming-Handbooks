## `const` и `readonly`

Оба используются для значений, которые **нельзя изменять**, но работают по-разному.

---

## `const`

```csharp
public const int Numbers = 4;
```

Особенности:
- значение задаётся **сразу при объявлении**;
- изменить потом нельзя;
- **обязательно** нужно инициализировать;
- автоматически является `static` (принадлежит классу).

Пример использования:

```csharp
Console.WriteLine(Customer.Numbers);
```

---

## `readonly`

```csharp
public readonly string Color;
```

Особенности:
- можно присвоить значение:
  - при объявлении,
  - или в **конструкторе**;
- после этого изменить нельзя;
- значение может быть **разным у разных объектов**.

Пример:

```csharp
public Customer(string color)
{
    Color = color;
}
```

Использование:

```csharp
Customer c1 = new Customer("Red");
Customer c2 = new Customer("Blue");
```

Получим:

```text
c1.Color = Red
c2.Color = Blue
```

---

## Главное отличие

`const`
→ одно фиксированное значение для всех объектов.

Пример:
```csharp
Math.PI
```

---

`readonly`
→ значение задаётся один раз, но может быть разным у каждого объекта.

Пример:
```csharp
ID клиента
Цвет машины
Дата создания
```
