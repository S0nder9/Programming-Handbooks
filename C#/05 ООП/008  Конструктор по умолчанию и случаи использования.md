## Конструктор по умолчанию и случаи использования

**Конструктор по умолчанию** (*default constructor*) — это конструктор **без параметров**.

Он вызывается, когда объект создаётся так:

```csharp
Customer1 newCustomer = new Customer1();
```

---

## Пример конструктора по умолчанию

```csharp
public Customer1()
{
    Name = "Default Name";
    Address = null;
    ContactNumber = "+";
}
```

Что делает:
- задаёт начальные значения по умолчанию;
- объект сразу готов к использованию.

---

## Использование

```csharp
Customer1 newCustomer = new Customer1();

Console.WriteLine(newCustomer.Name);
```

Результат:

```text
Default Name
```

---

Можно изменить свойства позже:

```csharp
newCustomer.Name = "John Doe";

Console.WriteLine(newCustomer.Name);
```

Результат:

```text
John Doe
```

---

## Когда используется конструктор по умолчанию?

### 1. Когда данные появятся позже

Например:

```csharp
Customer1 customer = new Customer1();

customer.Name = "Alex";
customer.Address = "London";
```

Сначала создаём объект, потом заполняем.

---

### 2. Для "пустого" объекта

Когда нужен объект-заготовка:

```csharp
Customer1 customer = new Customer1();
```

---

### 3. Для сериализации / библиотек

Многие библиотеки (JSON, Entity Framework и др.) требуют конструктор без параметров.

---

## Важно

Если написать свой конструктор:

```csharp
public Customer1(string name)
```

то **автоматический конструктор без параметров исчезает**.

Поэтому его нужно добавить вручную:

```csharp
public Customer1()
{
}
```

---

## Итог

```csharp
new Customer1();
```

→ вызовется конструктор без параметров.
