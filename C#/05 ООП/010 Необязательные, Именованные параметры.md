## Необязательные параметры (Optional Parameters)

Необязательные параметры позволяют задать **значение по умолчанию**, если пользователь не передал аргумент.

---

### В конструкторе

```csharp
public Customer(
    string name,
    string address,
    string contactNumber = "NA"
)
```

Если вызвать:

```csharp
Customer c = new Customer("John", "London");
```

то автоматически:

```text
contactNumber = "NA"
```

---

### В методе

```csharp
public void SetDetails(
    string name,
    string address,
    string contactNumber = "NA"
)
```

Вызов:

```csharp
customer.SetDetails("Alex", "Paris");
```

Результат:
```text
contactNumber = "NA"
```

---

## Правило

Необязательные параметры должны идти **в конце** списка параметров:

✅ правильно:

```csharp
void Test(string name, int age = 18)
```

❌ ошибка:

```csharp
void Test(int age = 18, string name)
```

---

# Именованные параметры (Named Parameters)

Позволяют передавать аргументы **по имени**, а не по порядку.

---

## Пример

```csharp
Customer customer = new Customer(
    name: "John",
    address: "London",
    contactNumber: "12345"
);
```

То же самое, что:

```csharp
Customer customer =
    new Customer("John", "London", "12345");
```

---

## Главное преимущество

Можно менять порядок:

```csharp
Customer customer = new Customer(
    address: "London",
    name: "John",
    contactNumber: "12345"
);
```

Это будет работать.

---

## Удобно с необязательными параметрами

Можно передать только нужные:

```csharp
Customer customer = new Customer(
    name: "John",
    address: "London"
);
```

`contactNumber` автоматически станет:

```text
NA
```

---

## Когда полезно?

Когда у метода много параметров:

```csharp
CreateUser(
    name: "Nikita",
    age: 20,
    country: "Sweden"
);
```

Так код читается намного лучше.
