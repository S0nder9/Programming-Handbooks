## Модификаторы `public`, `private`, `internal` и `protected`

**Модификаторы доступа** определяют, **откуда можно обращаться** к полям, свойствам, методам и классам.

---

## `public`

Доступен **отовсюду**.

```csharp
public string Name;
```

Можно обратиться из любого класса:

```csharp
customer.Name = "John";
```

Используется для:
- свойств,
- методов,
- классов.

---

## `private`

Доступен **только внутри своего класса**.

```csharp
private string _name;
```

Снаружи нельзя:

```csharp
customer._name; // Ошибка
```

Используется для:
- скрытия данных,
- инкапсуляции.

---

## `internal`

Доступен **только внутри текущей сборки (проекта)**.

```csharp
internal class Customer
{
}
```

Это значит:
- внутри этого проекта можно использовать;
- из другого проекта — нельзя.

Часто используется для классов.

---

## `protected`

Доступен:
- внутри своего класса;
- в **наследниках** (дочерних классах).

Пример:

```csharp
class Animal
{
    protected string Name;
}

class Dog : Animal
{
    public void Show()
    {
        Console.WriteLine(Name); // можно
    }
}
```

Из обычного объекта нельзя:

```csharp
dog.Name; // Ошибка
```

---

## Таблица

| Модификатор | Где доступен |
|------------|--------------|
| `public` | везде |
| `private` | только внутри класса |
| `internal` | только внутри проекта |
| `protected` | внутри класса + наследники |

---

## По умолчанию

Если не написать модификатор для поля:

```csharp
class Test
{
    int x;
}
```

то это будет:

```csharp
private int x;
```

(для полей класса по умолчанию `private`)
