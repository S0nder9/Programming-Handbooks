## Поля класса и пользовательский конструктор

**Поле класса** — это переменная, которая принадлежит объекту класса и хранит его данные (состояние).

Например, у автомобиля можно хранить **модель**.

---

## Поле класса

```csharp
private string _model = "";
```

Разберём:

- `private` — модификатор доступа:
  - поле доступно **только внутри класса** `Auto`;
  - другие классы не могут напрямую изменить `_model`.

Это называется **инкапсуляция** — защита данных внутри объекта.

---

## Пользовательский (параметризованный) конструктор

```csharp
public Auto(string model)
{
    _model = model;
    Console.WriteLine("An Obj of Car was created! " + model);
}
```

Это **конструктор с параметром**.

Когда создаём объект, нужно передать значение:

```csharp
Auto audi = new Auto("Audi");
```

Что происходит:
1. создаётся объект `Auto`;
2. в параметр `model` передаётся `"Audi"`;
3. значение сохраняется в поле `_model`.

То есть:

```text
_model = "Audi"
```

---

## Полный пример

```csharp
using System;

namespace ConsoleApp2
{
    internal class Auto
    {
        private string _model = "";

        public Auto(string model)
        {
            _model = model;
            Console.WriteLine("An Obj of Car was created! " + model);
        }
    }
}
```

---

## Использование

```csharp
class Program
{
    static void Main(string[] args)
    {
        Auto audi = new Auto("Audi");
        Auto bmw = new Auto("BMW");

        Console.ReadKey();
    }
}
```

Результат:
```text
An Obj of Car was created! Audi
An Obj of Car was created! BMW
```

---

## Почему нужен такой конструктор?

Без него пришлось бы отдельно задавать значение после создания объекта.

С ним можно сразу:

```csharp
Auto audi = new Auto("Audi");
```

Это удобнее и безопаснее.

---

## Идея

```text
Поле (_model) = хранит данные
Конструктор = заполняет данные при создании объекта
```
