## Использование нескольких конструкторов (перегрузка конструкторов)

В C# можно создавать **несколько конструкторов** в одном классе.

Это называется **перегрузка конструкторов** (*constructor overloading*).

Идея:
- один класс,
- несколько способов создать объект.

---

## Пример

```csharp
public class Customer1
{
    public string Name { get; set; }
    public string Address { get; set; }
    public string ContactNumber { get; set; }

    // Конструктор с тремя параметрами
    public Customer1(string name, string address, string contactNumber)
    {
        Name = name;
        Address = address;
        ContactNumber = contactNumber;
    }

    // Конструктор с одним параметром
    public Customer1(string name)
    {
        Name = name;
    }
}
```

---

## Как это работает

Можно создать объект двумя способами:

### 1. Полная информация

```csharp
Customer1 c1 = new Customer1(
    "Nikita",
    "Stockholm",
    "+46700000000"
);
```

Все свойства будут заполнены.

---

### 2. Только имя

```csharp
Customer1 c2 = new Customer1("Alex");
```

Результат:
- `Name = "Alex"`
- `Address = null`
- `ContactNumber = null`

---

## Как C# выбирает конструктор?

По **количеству и типу параметров**.

```csharp
new Customer1("Alex");
```

→ вызовется:

```csharp
Customer1(string name)
```

---

```csharp
new Customer1("Alex", "London", "123");
```

→ вызовется:

```csharp
Customer1(string name, string address, string contactNumber)
```

---

## Зачем это нужно?

Чтобы дать пользователю **разные способы создания объекта**.

Например:
- иногда есть все данные,
- иногда только часть.
