## Статические поля (`static fields`)

**Статическое поле** принадлежит **классу**, а не отдельному объекту.

Это значит:
- одно значение хранится **для всех объектов сразу**;
- все объекты используют одно и то же поле.

---

## Пример

```csharp
public class Customer
{
    public string Name { get; set; }

    public static int CustomerCount = 0;

    public Customer(string name)
    {
        Name = name;
        CustomerCount++;
    }
}
```

---

## Как работает?

Создаём объекты:

```csharp
Customer c1 = new Customer("John");
Customer c2 = new Customer("Alex");
Customer c3 = new Customer("Kate");
```

Каждый раз конструктор увеличивает:

```csharp
CustomerCount++;
```

---

## Получение значения

```csharp
Console.WriteLine(Customer.CustomerCount);
```

Результат:

```text
3
```

Почему `3`?
Потому что создано 3 объекта.

---

## Почему через имя класса?

Правильно:

```csharp
Customer.CustomerCount
```

Потому что поле принадлежит **классу**, а не объекту.

Не нужно:

```csharp
c1.CustomerCount
```

---

## Где используется?

Часто для:
- подсчёта количества объектов;
- хранения общих настроек;
- глобальных констант.

---

## Пример константы

```csharp
public class Settings
{
    public static string AppName = "My App";
}
```

Использование:

```csharp
Console.WriteLine(Settings.AppName);
```

Результат:

```text
My App
```

---

## Разница

Обычное поле:

```csharp
public string Name;
```

У каждого объекта своё:

```text
c1.Name = John
c2.Name = Alex
```

---

Статическое поле:

```csharp
public static int Count;
```

Одно на всех:

```text
Count = 3
```
