## Статические методы (`static`)

**Статический метод** принадлежит **самому классу**, а не объекту.

Поэтому его можно вызвать **без создания объекта** (если метод не требует объект как параметр).

---

## Пример статического метода

```csharp
public static void DisplayCustomerInfo(Customer customer)
{
    Console.WriteLine($"Name: {customer.Name}");
    Console.WriteLine($"Address: {customer.Address}");
    Console.WriteLine($"Contact Number: {customer.ContactNumber}");
}
```

Разберём:
- `static` — метод принадлежит классу `Customer`;
- `void` — ничего не возвращает;
- принимает объект `Customer` как параметр.

---

## Правильный вызов

```csharp
Customer customer =
    new Customer("John Doe", "123 Main St", "555-1234");

Customer.DisplayCustomerInfo(customer);
```

Результат:

```text
Name: John Doe
Address: 123 Main St
Contact Number: 555-1234
```

Вызываем через **имя класса**:

```csharp
Customer.DisplayCustomerInfo(...)
```

---

## Неправильный вызов

```csharp
customer.DisplayCustomerInfo(customer);
```

Так вызывать **не нужно**, потому что метод статический.

Он не принадлежит объекту `customer`, а принадлежит классу `Customer`.

---

## Когда использовать `static`?

Когда метод:
- не использует данные конкретного объекта (`this`);
- выполняет общую логику;
- полезен для всех объектов.

Например:
- `Math.Sqrt()`
- `Console.WriteLine()`

Это тоже статические методы:

```csharp
Math.Sqrt(25);
Console.WriteLine("Hello");
```

Мы не создаём:

```csharp
Math math = new Math(); // нельзя
```

---

## Разница

Обычный метод:

```csharp
customer.Drive();
```

→ вызывается на объекте.

---

Статический метод:

```csharp
Customer.DisplayCustomerInfo(customer);
```

→ вызывается на классе.
