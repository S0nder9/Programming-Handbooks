## Смешивание `double` и `int` при вычислениях

В C# можно выполнять операции между разными числовыми типами, например `int` и `double`.

При этом C# автоматически приводит результат к **более точному типу**.

---

## `int + double`

```csharp
int a = 5;
double b = 2.5;

double result = a + b;

Console.WriteLine(result);
```

Результат:
```text
7.5
```

Почему?
- `int` (`5`) автоматически преобразуется в `5.0`
- получается:

```text
5.0 + 2.5 = 7.5
```

Это называется **неявное преобразование типов** (*implicit conversion*).

---

## `int / int`

Важно: если делим два `int`, результат тоже будет `int`.

```csharp
int a = 5;
int b = 2;

Console.WriteLine(a / b);
```

Результат:
```text
2
```

Почему?
Потому что дробная часть **отбрасывается**:

```text
2.5 → 2
```

---

## Как получить дробный результат?

Нужно хотя бы одно число сделать `double`:

```csharp
Console.WriteLine(5.0 / 2);
```

или:

```csharp
Console.WriteLine((double)5 / 2);
```

Результат:
```text
2.5
```

---

## Явное преобразование (`casting`)

Можно вручную преобразовать тип:

```csharp
double x = 9.8;
int y = (int)x;

Console.WriteLine(y);
```

Результат:
```text
9
```

Дробная часть отбрасывается.

---

## Потеря точности

```csharp
double pi = 3.14159;
int whole = (int)pi;
```

Получим:

```text
3
```

Информация после запятой потеряна.

---

## Пример смешивания

```csharp
int apples = 3;
double price = 1.99;

double total = apples * price;

Console.WriteLine(total);
```

Результат:
```text
5.97
```

`3` автоматически превращается в `3.0`.

---

## Таблица преобразований

| Выражение | Результат |
|----------|-----------|
| `5 + 2.5` | `7.5` (`double`) |
| `5 / 2` | `2` (`int`) |
| `5.0 / 2` | `2.5` (`double`) |
| `(int)3.9` | `3` |
| `(double)5 / 2` | `2.5` |

---

## Правило

Если участвует `double`, результат обычно будет `double`.

```csharp
int + double -> double
int * double -> double
int / double -> double
```
