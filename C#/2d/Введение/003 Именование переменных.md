## Именование переменных

Правильное именование переменных делает код **понятным и читаемым**.

Переменная должна отражать **смысл данных**, которые хранит.

---

## Хорошие имена

```csharp
string firstName;
int age;
double totalPrice;
bool isActive;
```

Сразу понятно, что хранится.

---

## Плохие имена

```csharp
string a;
int x;
double q;
```

Непонятно, что означают переменные.

---

## Стиль `camelCase`

Для переменных и параметров в C# используется **camelCase**:

```csharp
string firstName;
int totalAmount;
```

Правило:
- первое слово с маленькой буквы;
- каждое следующее — с большой.

---

## Поля класса (`private`)

Обычно используют `_camelCase`:

```csharp
private string _name;
private int _age;
```

Подчёркивание показывает, что это поле класса.

---

## Свойства и методы

Используют **PascalCase**:

```csharp
public string Name { get; set; }

public void Drive()
{
}
```

Каждое слово с большой буквы.

---

## Константы

Часто тоже `PascalCase`:

```csharp
public const int MaxUsers = 100;
```

---

## Для `bool`

Хорошо начинать с:
- `is`
- `has`
- `can`

Примеры:

```csharp
bool isActive;
bool hasAccess;
bool canDrive;
```

Сразу понятно, что это логическое значение (`true/false`).

---

## Правила

✅ Используй понятные имена:
```csharp
customerName
```

❌ Не используй:
```csharp
cn
```

---

## Идея

```text
Переменная должна объяснять себя сама.
```
