## Ссылочные типы

В C# массивы относятся к **ссылочным типам** (*reference types*).

Это означает, что переменная хранит **не сами данные**, а **ссылку на область памяти**, где эти данные находятся.

---

## Пример

```csharp
int[] array = new int[3];
int[] tempArray = new int[5];

array = tempArray;
```

После присваивания:

```csharp
array = tempArray;
```

обе переменные указывают на **один и тот же массив** в памяти.

Старый массив размером 3 становится недоступным и позже будет удалён сборщиком мусора (Garbage Collector).

---

## Что происходит дальше?

```csharp
array[0] = 5;
```

Так как `array` и `tempArray` ссылаются на один объект, изменение через одну переменную видно через другую.

---

## Результат

```csharp
Console.WriteLine(tempArray[0]);
```

Вывод:

```text
5
```

---

## Схема в памяти

До присваивания:

```text
array     ---> [0][0][0]

tempArray ---> [0][0][0][0][0]
```

После:

```text
array -----\
            ---> [0][0][0][0][0]
tempArray--/
```

После:

```csharp
array[0] = 5;
```

```text
array -----\
            ---> [5][0][0][0][0]
tempArray--/
```

---

## Копируется не массив, а ссылка

```csharp
int[] a = { 1, 2, 3 };
int[] b = a;
```

Здесь НЕ создаётся новый массив.

Копируется только ссылка:

```text
a ----\
       ---> [1][2][3]
b ----/
```

Поэтому:

```csharp
b[0] = 100;
```

изменит данные и для `a`:

```csharp
Console.WriteLine(a[0]); // 100
```

---

## Отличие от значимых типов

Для `int` копируется само значение:

```csharp
int a = 5;
int b = a;

b = 10;

Console.WriteLine(a); // 5
```

`a` не изменился, потому что `int` — **значимый тип** (*value type*).

---

## Итог

- `int`, `double`, `bool`, `char` — значимые типы.
- Массивы, классы, строки — ссылочные типы.
- При присваивании ссылочного типа копируется **ссылка**, а не объект.
- Изменение объекта через одну переменную будет видно через все ссылки на этот объект.
