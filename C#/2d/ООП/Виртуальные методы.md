## Виртуальные методы

Виртуальные методы позволяют наследникам изменять реализацию метода базового класса.

Используются для реализации полиморфизма:

```text
Один метод → разное поведение у разных объектов.
```

---

## Ключевые слова

```csharp
virtual
```

Метод можно переопределить в наследнике.

---

```csharp
override
```

Переопределяет реализацию базового метода.

---

```csharp
base
```

Позволяет вызвать реализацию родительского класса.

---

## Базовый класс

```csharp
class NPS
{
    public virtual void ShowDesc()
    {
        Console.WriteLine("NPC");
    }
}
```

Метод объявлен как `virtual`, поэтому наследники могут изменить его поведение.

---

## Переопределение метода

```csharp
class Farmer : NPS
{
    public override void ShowDesc()
    {
        base.ShowDesc();
        Console.WriteLine("Фермер");
    }
}
```

Здесь:

```csharp
base.ShowDesc();
```

сначала вызывает метод родителя:

```text
NPC
```

а затем:

```text
Фермер
```

---

## Полное переопределение

```csharp
class Knight : NPS
{
    public override void ShowDesc()
    {
        Console.WriteLine("Рыцарь");
    }
}
```

Здесь вызова `base.ShowDesc()` нет.

Поэтому родительская логика полностью заменяется.

---

## Наследник без переопределения

```csharp
class Child : NPS
{
}
```

Так как метод не переопределён, используется версия из `NPS`.

---

## Использование

```csharp
NPS[] chars =
{
    new NPS(),
    new Farmer(),
    new Knight(),
    new Child()
};
```

---

```csharp
foreach (var item in chars)
{
    item.ShowDesc();
}
```

Результат:

```text
NPC

NPC
Фермер

Рыцарь

NPC
```

---

## Почему это работает?

Переменная имеет тип:

```csharp
NPS
```

но во время выполнения CLR смотрит на реальный объект:

```csharp
Farmer
Knight
Child
```

и вызывает нужную реализацию метода.

Это и есть полиморфизм.

---

## Без virtual

Если убрать:

```csharp
virtual
```

и

```csharp
override
```

то при обращении через тип `NPS` всегда вызывался бы метод базового класса.

---

## Кратко

```text
virtual  → метод можно переопределить

override → новая реализация метода

base     → вызов реализации родителя
```

Виртуальные методы позволяют объектам одного базового типа выполнять разные действия в зависимости от их реального класса.
