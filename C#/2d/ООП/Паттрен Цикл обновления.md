## Паттерн «Цикл обновления» (Update Loop)

Паттерн **Update Loop** — это бесконечный цикл, в котором программа регулярно обновляет состояние объектов.

Часто используется в:
- играх;
- симуляциях;
- системах реального времени.

---

## Идея

Вместо того чтобы вручную вызывать методы каждого объекта:

```csharp
walker.Update();
jumper.Update();
```

мы собираем их в коллекцию и обновляем в цикле.

---

## Базовый класс

```csharp
class Behavior
{
    public virtual void Update()
    {
    }
}
```

Каждый объект поведения должен реализовать свой метод обновления.

---

## Конкретные поведения

```csharp
class Walker : Behavior
{
    public override void Update()
    {
        Console.WriteLine("GO");
    }
}
```

---

```csharp
class Jumper : Behavior
{
    public override void Update()
    {
        Console.WriteLine("Jump");
    }
}
```

---

## Массив объектов

```csharp
Behavior[] behav =
{
    new Walker(),
    new Jumper()
};
```

Здесь разные объекты хранятся в одном массиве благодаря полиморфизму.

---

## Цикл обновления

```csharp
while (true)
{
    foreach (var item in behav)
    {
        item.Update();
    }
}
```

Каждую итерацию цикла все объекты получают возможность обновить своё состояние.

---

## Что происходит?

Первая итерация:

```text
GO
Jump
```

Вторая итерация:

```text
GO
Jump
```

И так бесконечно.

---

## Thread.Sleep

```csharp
Thread.Sleep(1000);
```

Приостанавливает выполнение потока на 1000 мс (1 секунду).

В твоём коде задержка стоит внутри `foreach`:

```csharp
foreach (var item in behav)
{
    item.Update();
    Thread.Sleep(1000);
}
```

Поэтому получится:

```text
GO
(1 секунда)

Jump
(1 секунда)

GO
(1 секунда)

Jump
...
```

---

Обычно в играх задержку ставят после обновления всех объектов:

```csharp
while (true)
{
    foreach (var item in behav)
    {
        item.Update();
    }

    Thread.Sleep(1000);
}
```

Тогда обновление считается одним кадром.

Проверка подключений

## Кратко

```text
Update Loop = бесконечный цикл обновления объектов.

while (true)
{
    object.Update();
}
```

Каждый объект сам решает, что делать при очередном обновлении.
