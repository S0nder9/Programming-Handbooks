## Статические члены и классы

**Статические (`static`) члены принадлежат самому классу, а не объекту.**

Поэтому для работы с ними не нужно создавать объект.

---

## Статическое поле

```csharp
public static int Identifications;
```

Общее для всех объектов класса.

Использование:

```csharp
User.Identifications = 10;
```

Доступ происходит через имя класса:

```csharp
Класс.Поле
```

---

## Обычное поле

```csharp
public int Ident;
```

У каждого объекта своё значение.

---

## Пример

```csharp
User.Identifications = 10;

User user1 = new User();
User user2 = new User();
```

Конструктор:

```csharp
public User()
{
    Ident = ++Identifications;
}
```

Создание объектов:

```text
Identifications = 10

user1:
Identifications = 11
Ident = 11

user2:
Identifications = 12
Ident = 12
```

Вывод:

```text
11
12
```

---

## Статические методы

```csharp
public static int GetSelPerDay(int h)
{
    return MenHPrice * h;
}
```

Вызываются через класс:

```csharp
User.GetSelPerDay(8);
```

---

### Важное ограничение

Статический метод не может обращаться к нестатическим полям:

```csharp
public int Ident;
```

Нельзя:

```csharp
public static void Test()
{
    Console.WriteLine(Ident); // Ошибка
}
```

Потому что объекта может вообще не существовать.

---

### Можно обращаться к static

```csharp
public static int MenHPrice;
```

Поэтому:

```csharp
public static int GetSelPerDay(int h)
{
    return MenHPrice * h;
}
```

работает корректно.

---

## Вызов статического метода из другого статического метода

```csharp
public static int GetSelPerMonth(int days)
{
    return GetSelPerDay(2) * days;
}
```

Статические методы могут вызывать другие статические методы без создания объекта.

---

## Когда использовать static?

### Счётчики объектов

```csharp
public static int UserCount;
```

---

### Настройки приложения

```csharp
public static string AppName = "My App";
```

---

### Утилиты

```csharp
Math.Sqrt(25);
```

Класс `Math` состоит почти полностью из статических методов.

---

## Статический класс

Если класс содержит только статические члены:

```csharp
static class MathHelper
{
    public static int Sum(int a, int b)
    {
        return a + b;
    }
}
```

Создать объект нельзя:

```csharp
MathHelper helper = new MathHelper(); // Ошибка
```

Использование:

```csharp
MathHelper.Sum(2, 3);
```

---

## Кратко

```text
Обычное поле:
У каждого объекта своё значение.

Статическое поле:
Одно значение на весь класс.
```

```csharp
User user1 = new User();
User user2 = new User();
```

```text
user1.Ident      → своё значение
user2.Ident      → своё значение

User.Identifications → общее значение
```

---

### Правило

```text
instance-члены → через объект

static-члены → через класс
```

Примеры:

```csharp
user.ShowInfo();

User.GetSelPerDay(8);

User.Identifications = 100;
```
