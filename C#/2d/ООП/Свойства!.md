## Свойства (Properties)

Свойства позволяют безопасно предоставлять доступ к данным объекта.

Без свойств часто приходится писать отдельные методы для получения значений:

```csharp
public int GetPosX()
{
    return X;
}

public int GetPosY()
{
    return Y;
}
```

Использование:

```csharp
render.Draw(player.GetPosX(), player.GetPosY());
```

Такой код работает, но становится неудобным, когда данных много.

---

## Свойства

Свойство выглядит как обычное поле:

```csharp
player.X
```

но внутри может содержать дополнительную логику.

---

### Свойство с `get` и `set`

```csharp
private int _x;

public int X
{
    get
    {
        return _x;
    }
    private set
    {
        if (value > 0 && value < 100)
            _x = value;
    }
}
```

#### Что делают аксессоры?

```csharp
get
```

Вызывается при чтении значения:

```csharp
Console.WriteLine(player.X);
```

---

```csharp
set
```

Вызывается при присвоении:

```csharp
player.X = 50;
```

Новое значение находится в специальной переменной:

```csharp
value
```

---


## Ограничение доступа к изменению

```csharp
public int X
{
    get;
    private set;
}
```

Значение можно:

```csharp
Console.WriteLine(player.X); // можно
```

Но нельзя:

```csharp
player.X = 50; // ошибка
```

Потому что `set` доступен только внутри класса.

---

## Автосвойства

Если дополнительная логика не нужна:

```csharp
public int Y { get; private set; }
```

Компилятор сам создаст скрытое поле.

Это короче, чем:

```csharp
private int _y;

public int Y
{
    get { return _y; }
    private set { _y = value; }
}
```

---

## Инициализация через конструктор

```csharp
public Player(int x, int y)
{
    X = x;
    Y = y;
}
```

Даже если `set` приватный, конструктор класса может его использовать.

---

## Зачем нужны свойства?

- скрывают внутренние поля;
- позволяют проверять данные;
- контролируют чтение и запись;
- делают код удобнее и безопаснее.

---

## Кратко

```text
Поле      → хранит данные
Свойство  → управляет доступом к данным

get → чтение
set → запись

value → новое присваиваемое значение
```

Обычно поля делают `private`, а доступ к ним предоставляют через свойства.
