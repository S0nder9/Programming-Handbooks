## Типизация

Типизация — это система правил, которая определяет, какие данные можно хранить в переменных и какие операции над ними выполнять.

Например:

```csharp
int age = 18;
string name = "Alex";
```

Компилятор знает тип каждой переменной и может проверить код ещё до запуска программы.

---

### 1. Система типов

В C# типы должны быть известны на этапе компиляции.

Компилятор проверяет:

- можно ли присвоить значение переменной;
- существует ли вызываемый метод;
- допустима ли операция между типами.

Пример:

```csharp
int age = 18;
age = "Hello"; // Ошибка компиляции
```

---

### 2. Контракты

Типы выступают в роли контракта между частями программы.

Например:

```csharp
static int Sum(int a, int b)
{
    return a + b;
}
```

Контракт метода:

```text
Принимает два int
Возвращает int
```

Нельзя вызвать так:

```csharp
Sum("1", "2"); // Ошибка
```

---

### 3. Помощь при рефакторинге

Если изменить тип или сигнатуру метода, компилятор покажет все места, которые нужно исправить.

Например:

```csharp
void Print(string text)
```

изменили на

```csharp
void Print(int text)
```

Компилятор сразу найдёт неверные вызовы.

---

### 4. Поддержка IDE

Благодаря типизации среда разработки может:

- подсказывать методы;
- показывать документацию;
- автоматически дописывать код;
- находить ошибки ещё до запуска.

Пример:

```csharp
string text = "Hello";
text.
```

IDE покажет все доступные методы строки.

---

### 5. Unit-тестирование

Типизация упрощает написание тестов.

Если метод принимает:

```csharp
Customer customer
```

то тест обязан передать объект `Customer`, а не произвольные данные.

Это уменьшает количество ошибок.

---

## Плюсы строгой типизации

- меньше ошибок во время выполнения;
- больше ошибок находится при компиляции;
- удобнее поддерживать большие проекты;
- лучше работают инструменты разработки;
- код проще читать и понимать.
