## Статический конструктор

**Статический конструктор** используется для инициализации статических полей класса.

Он выполняется автоматически **один раз за всё время работы программы**.

---

## Объявление

```csharp
class MyClass
{
    public static float StaticField;

    static MyClass()
    {
        StaticField = 10;
        Console.WriteLine("s1");
    }
}
```

Особенности:

- имеет ключевое слово `static`;
- не имеет параметров;
- не имеет модификаторов доступа (`public`, `private` и т.д.);
- вызывается автоматически.

---

## Когда выполняется?

Перед первым использованием класса:

```csharp
MyClass.StaticField
```

или

```csharp
new MyClass()
```

---

## Пример

```csharp
Console.WriteLine("Hello wo");

MyClass insta = new MyClass();

Console.WriteLine(MyClass.StaticField);
```

---

### Что произойдёт?

1. Запустится программа

```text
Hello wo
```

2. Выполнится статический конструктор

```text
s1
```

3. Создастся объект

```text
DEF
```

4. Выведется значение поля

```text
10
```

---

### Результат

```text
Hello wo
s1
DEF
10
```

---

## Выполняется только один раз

```csharp
MyClass a = new MyClass();
MyClass b = new MyClass();
MyClass c = new MyClass();
```

Вывод:

```text
s1
DEF
DEF
DEF
```

Статический конструктор выполнился только при первом обращении к классу.

---

## Отличие от обычного конструктора

### Обычный конструктор

```csharp
public MyClass()
{
    Console.WriteLine("DEF");
}
```

Вызывается при каждом:

```csharp
new MyClass()
```

---

### Статический конструктор

```csharp
static MyClass()
{
}
```

Вызывается один раз для всего класса.

---

## Для чего используется?

### Инициализация статических полей

```csharp
static MyClass()
{
    StaticField = 10;
}
```

---

### Загрузка конфигурации

```csharp
static Settings()
{
    LoadConfig();
}
```

---

### Подготовка общих данных

```csharp
static Database()
{
    ConnectionString = "...";
}
```

---

## Аналог без конструктора

Можно написать:

```csharp
public static float StaticField = 10;
```

Тогда статический конструктор не нужен.

Но если логика сложнее:

```csharp
static MyClass()
{
    StaticField = ReadValue();
}
```

используют статический конструктор.

---

## Кратко

```text
Статический конструктор:
- принадлежит классу;
- вызывается автоматически;
- выполняется один раз;
- используется для инициализации static-полей.
```

```csharp
static MyClass()
{
    StaticField = 10;
}
```
