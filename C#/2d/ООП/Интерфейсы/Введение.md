## Интерфейсы

**Интерфейс** — это контракт, который описывает, что объект умеет делать, но не содержит реализации.

Интерфейс отвечает на вопрос:

```text
Что объект умеет делать?
```

а не

```text
Чем объект является?
```

---

## Объявление интерфейса

```csharp
interface IMovable
{
    void Move();
    void ShowMS();
}
```

Здесь описаны два метода, но их реализация отсутствует.

---

## Ещё один интерфейс

```csharp
interface IBurn
{
    void Burn();
}
```

---

## Реализация интерфейса

```csharp
class Car : IMovable
{
    public void Move()
    {
        Console.WriteLine("Car moves");
    }

    public void ShowMS()
    {
        Console.WriteLine("100 km/h");
    }
}
```

Класс обязан реализовать все методы интерфейса.

---

## Несколько интерфейсов

Класс может реализовывать несколько интерфейсов одновременно.

```csharp
class Car : Vehicle, IMovable, IBurn
{
    public void Move()
    {
    }

    public void ShowMS()
    {
    }

    public void Burn()
    {
    }
}
```

---

## Наследование от класса + интерфейсы

В C# можно:

```csharp
class Car : Vehicle, IMovable, IBurn
```

То есть:

- наследоваться только от одного класса;
- реализовывать сколько угодно интерфейсов.

---

## Использование интерфейса

```csharp
IMovable car = new Car();

car.Move();
```

Переменная знает только про методы интерфейса:

```csharp
Move()
ShowMS()
```

---

Поэтому:

```csharp
car.Burn();
```

будет ошибкой компиляции.

Для переменной типа `IMovable` метода `Burn()` не существует.

---

## Полиморфизм через интерфейсы

```csharp
IMovable[] movables =
{
    new Car(),
    new Human()
};
```

---

```csharp
foreach (IMovable movable in movables)
{
    movable.Move();
}
```

Каждый объект реализует движение по-своему.

---


## Интерфейс vs Абстрактный класс

### Интерфейс

```csharp
interface IFlyable
{
    void Fly();
}
```

Описывает возможность:

```text
умеет летать
умеет двигаться
умеет стрелять
```

---

### Абстрактный класс

```csharp
abstract class Animal
{
    public void Sleep()
    {
    }
}
```

Описывает общую сущность и может содержать реализацию.

---

## Кратко

```text
Интерфейс = контракт

interface IMovable
{
    void Move();
}
```

- не хранит состояние;
- описывает возможности объекта;
- класс обязан реализовать все методы интерфейса;
- один класс может реализовывать много интерфейсов.
