## Полиморфизм

**Полиморфизм** — это возможность работать с разными объектами через общий интерфейс или базовый класс.

Часто говорят:

```text
Один интерфейс — множество реализаций.
```

---

## Пример

Есть общий интерфейс:

```csharp
interface IAnimal
{
    void MakeSound();
}
```

Разные реализации:

```csharp
class Dog : IAnimal
{
    public void MakeSound()
    {
        Console.WriteLine("Woof");
    }
}

class Cat : IAnimal
{
    public void MakeSound()
    {
        Console.WriteLine("Meow");
    }
}
```

---

## Использование

```csharp
IAnimal animal = new Dog();
animal.MakeSound();
```

Результат:

```text
Woof
```

Теперь:

```csharp
animal = new Cat();
animal.MakeSound();
```

Результат:

```text
Meow
```

Переменная имеет один тип:

```csharp
IAnimal
```

но поведение зависит от конкретного объекта.

---

## Полиморфизм через наследование

```csharp
class Animal
{
    public virtual void MakeSound()
    {
        Console.WriteLine("...");
    }
}

class Dog : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("Woof");
    }
}
```

Использование:

```csharp
Animal animal = new Dog();
animal.MakeSound();
```

Результат:

```text
Woof
```

Вызывается версия метода объекта `Dog`, а не `Animal`.

---

## Зачем нужен?

Без полиморфизма:

```csharp
Dog dog = new Dog();
Cat cat = new Cat();

dog.MakeSound();
cat.MakeSound();
```

С полиморфизмом:

```csharp
List<IAnimal> animals = new List<IAnimal>()
{
    new Dog(),
    new Cat()
};

foreach (IAnimal animal in animals)
{
    animal.MakeSound();
}
```

Код работает с любыми животными через общий интерфейс.

---

## Кратко

```text
Инкапсуляция  → защита данных
Абстракция    → скрытие деталей
Наследование  → получение функциональности
Полиморфизм   → один интерфейс, много реализаций
```

Полиморфизм отвечает на вопрос:

```text
Как работать с разными объектами одинаковым способом?
```
