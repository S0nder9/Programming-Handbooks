## Поля и модификаторы доступа

**Поле** — это переменная, которая принадлежит объекту класса и хранит его состояние.

```csharp
class Car
{
    public string Name;
}
```

Каждый объект `Car` будет иметь своё поле `Name`.

---

## Модификатор `public`

```csharp
public string Name;
```

`public` означает, что поле доступно из любого места программы.

Поэтому в `Main` можно написать:

```csharp
Car fer = new Car();

fer.Name = "Ferrari";
```

---

## Модификатор `private`

```csharp
private string _age;
```

`private` означает, что поле доступно только внутри самого класса.

Например:

```csharp
class Car
{
    private string _age;
}
```

Такой код вызовет ошибку:

```csharp
Car car = new Car();

car._age = "10"; // Ошибка
```

Потому что поле закрыто от внешнего кода.

---

## Почему используют `private`?

Чтобы защитить внутреннее состояние объекта.

Обычно данные хранятся в закрытых полях, а доступ к ним предоставляется через свойства:

```csharp
class Car
{
    private string _name;

    public string Name
    {
        get => _name;
        set => _name = value;
    }
}
```

Это называется **инкапсуляцией**.

---

## Ссылочное поведение объектов

```csharp
Car fer = new Car();
fer.Name = "s";

Car moz = new Car();
moz = fer;

moz.Name = "s2";

Console.WriteLine(fer.Name);
```

Результат:

```text
s2
```

Почему?

После строки:

```csharp
moz = fer;
```

обе переменные ссылаются на один объект.

Схема:

```text
fer ────┐
        ▼
      Car
        ▲
moz ────┘
```

Изменение через `moz` видно через `fer`.

---

## Соглашение об именовании

Часто закрытые поля называют с подчёркиванием:

```csharp
private string _name;
private int _speed;
private bool _isRunning;
```

Это помогает сразу отличать поля от локальных переменных и свойств.

---

## Кратко

```text
public  → доступен отовсюду
private → доступен только внутри класса

Поле хранит данные объекта.

Переменные классов содержат ссылки на объекты,
поэтому несколько переменных могут ссылаться на один объект.
```
