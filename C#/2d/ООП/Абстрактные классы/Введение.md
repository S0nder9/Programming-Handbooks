## Абстрактные классы

**Абстрактный класс** — это класс-шаблон (чертёж), на основе которого создаются другие классы.

Его задача — описать общее поведение и данные для группы объектов.

---

## Зачем нужен?

Например, есть разные виды транспорта:

```text
Автомобиль
Поезд
Самолёт
Корабль
```

У всех есть:

```text
Скорость
Движение
```

Поэтому общую логику можно вынести в базовый класс.

---

## Абстрактный класс

```csharp
abstract class Vehicle
{
    protected float Speed;

    public abstract void Move();

    public float GetCurSpeed()
    {
        return Speed;
    }
}
```

Ключевое слово:

```csharp
abstract
```

означает, что создать объект этого класса нельзя:

```csharp
Vehicle vehicle = new Vehicle(); // Ошибка
```

---

## Абстрактный метод

```csharp
public abstract void Move();
```

Такой метод:

- не имеет реализации;
- обязан быть реализован в наследниках.

По сути это требование:

```text
Каждый транспорт должен уметь двигаться.
```

---

## Реализация в наследниках

### Автомобиль

```csharp
class Car : Vehicle
{
    public override void Move()
    {
        Console.WriteLine("Car moves");
    }
}
```

---

### Поезд

```csharp
class Train : Vehicle
{
    public override void Move()
    {
        Console.WriteLine("Train moves");
    }
}
```

---

## Использование

```csharp
Vehicle[] vehicles =
{
    new Car(),
    new Train()
};
```

---

```csharp
foreach (var item in vehicles)
{
    item.Move();
}
```

Результат:

```text
Car moves
Train moves
```

Хотя переменная имеет тип:

```csharp
Vehicle
```

вызывается реализация конкретного объекта.

Это пример полиморфизма.

---

## Отличие от интерфейса

### Интерфейс

```csharp
interface IMovable
{
    void Move();
}
```

Описывает только контракт.

---

### Абстрактный класс

```csharp
abstract class Vehicle
{
    protected float Speed;

    public abstract void Move();

    public float GetCurSpeed()
    {
        return Speed;
    }
}
```

Может содержать:

- поля;
- свойства;
- обычные методы;
- абстрактные методы.

---

## Когда использовать?

### Интерфейс

Когда нужно описать возможность:

```text
умеет двигаться
умеет летать
умеет плавать
```

---

### Абстрактный класс

Когда есть общая логика и состояние:

```text
Vehicle
Animal
Character
Employee
```

---

## Кратко

```text
Абстрактный класс = неполный класс-шаблон.

abstract class Vehicle
{
    public abstract void Move();
}
```

- нельзя создать объект;
- может содержать поля и методы;
- может содержать абстрактные методы;
- наследники обязаны реализовать abstract-методы.
