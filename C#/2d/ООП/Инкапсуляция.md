## Инкапсуляция

**Инкапсуляция** — это сокрытие внутреннего состояния объекта и предоставление контролируемого доступа к нему.

Основная цель — защитить данные объекта от некорректного изменения извне.

---

## Пример без инкапсуляции

```csharp
class Player
{
    public int Health;
}

Player player = new Player();

player.Health = -1000;
```

Любой код может записать в `Health` недопустимое значение.

---

## Пример с инкапсуляцией

```csharp
class Player
{
    private int _health;

    public int Health
    {
        get => _health;
        set
        {
            if (value >= 0)
                _health = value;
        }
    }
}
```

Теперь:

```csharp
player.Health = -1000;
```

не изменит здоровье игрока.

---

## Инструменты инкапсуляции

### Поля `private`

Скрывают данные внутри класса.

```csharp
private int _health;
```

---

### Свойства (`get` / `set`)

Позволяют контролировать чтение и изменение данных.

```csharp
public int Health
{
    get => _health;
    set => _health = value;
}
```

---

### Методы

Иногда объект вообще не даёт менять данные напрямую.

```csharp
class BankAccount
{
    private decimal _balance;

    public void Deposit(decimal amount)
    {
        if (amount > 0)
            _balance += amount;
    }
}
```

Изменение баланса возможно только через метод.

---

## Зачем нужна инкапсуляция?

- защита данных от неправильного использования;
- контроль изменений состояния объекта;
- уменьшение количества ошибок;
- возможность менять внутреннюю реализацию без изменения внешнего кода.

---

## Кратко

```text
Инкапсуляция =
скрываем данные (private)
+
предоставляем безопасный доступ (public свойства и методы).
```

Она отвечает на вопрос:

```text
Как защитить объект от неправильного использования?
```
