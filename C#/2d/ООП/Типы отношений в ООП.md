## Типы отношений в ООП

### 1. Генерализация (Generalization)

Отношение **"является" (is-a)**.

Один класс является более частным случаем другого класса.

В C# реализуется через **наследование**.

```csharp
class Animal
{
}

class Dog : Animal
{
}
```

Здесь:

```text
Dog является Animal
```

---

### 2. Реализация (Realization)

Класс реализует интерфейс и обязуется предоставить реализацию его методов.

```csharp
interface IDriveable
{
    void Drive();
}

class Car : IDriveable
{
    public void Drive()
    {
        Console.WriteLine("Driving");
    }
}
```

Здесь:

```text
Car реализует IDriveable
```

---

### 3. Ассоциация (Association)

Один объект знает о другом объекте и может с ним взаимодействовать.

Объекты существуют независимо друг от друга.

```csharp
class Driver
{
}

class Car
{
    public Driver Driver { get; set; }
}
```

Здесь:

```text
Машина связана с водителем
```

Но если удалить машину, водитель останется существовать.

---

### 4. Композиция (Composition)

Отношение **"состоит из" (has-a)**.

Один объект полностью владеет другим объектом.

Если уничтожается основной объект, уничтожаются и его части.

```csharp
class Engine
{
}

class Car
{
    private Engine _engine;

    public Car()
    {
        _engine = new Engine();
    }
}
```

Здесь:

```text
Car состоит из Engine
```

Двигатель создаётся внутри автомобиля и отдельно не существует.

---

### 5. Агрегация (Aggregation)

Похожа на композицию, но вложенный объект может существовать самостоятельно.

```csharp
class Engine
{
}

class Car
{
    public Engine Engine { get; }

    public Car(Engine engine)
    {
        Engine = engine;
    }
}
```

Использование:

```csharp
Engine engine = new Engine();
Car car = new Car(engine);
```

Здесь:

```text
Car использует Engine
```

Но двигатель может существовать без автомобиля.

---

## Краткая таблица

| Отношение | Смысл | Пример |
|------------|--------|---------|
| Generalization | является (is-a) | Dog → Animal |
| Realization | реализует интерфейс | Car → IDriveable |
| Association | знает о другом объекте | Car ↔ Driver |
| Composition | состоит из | Car → Engine |
| Aggregation | использует готовый объект | Car → Engine |

---

### Как быстро запомнить

```text
Generalization → наследование
Realization    → интерфейсы
Association    → просто связь
Composition    → сильное владение
Aggregation    → слабое владение
```
