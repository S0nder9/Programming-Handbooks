## Абстракция

**Абстракция** — это выделение главного и сокрытие лишних деталей реализации.

Например, когда ты нажимаешь кнопку в приложении, тебе не нужно знать, какой код выполняется внутри.

---

## 2 вида абстракции в C#

### 1. Абстрактные классы

Позволяют описать общую модель объекта и частично реализовать её.

```csharp
abstract class Animal
{
    public void Sleep()
    {
        Console.WriteLine("Sleeping...");
    }

    public abstract void MakeSound();
}
```

Наследник обязан реализовать абстрактный метод:

```csharp
class Dog : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("Woof");
    }
}
```

Использование:

```csharp
Dog dog = new Dog();
dog.MakeSound();
dog.Sleep();
```

---

### 2. Интерфейсы

Описывают только контракт: что объект умеет делать.

```csharp
interface IFlyable
{
    void Fly();
}
```

Класс обязан реализовать все методы интерфейса:

```csharp
class Bird : IFlyable
{
    public void Fly()
    {
        Console.WriteLine("Flying...");
    }
}
```

---

## Разница

### Абстрактный класс

Может содержать:

```csharp
- поля
- свойства
- обычные методы
- абстрактные методы
```

То есть может хранить общую реализацию.

---

### Интерфейс

Описывает только возможности объекта:

```csharp
IFlyable
IDriveable
IPrintable
```

Интерфейс отвечает на вопрос:

```text
Что объект умеет делать?
```

---

## Когда использовать?

### Абстрактный класс

Когда есть общая базовая логика:

```text
Animal
 ├─ Dog
 ├─ Cat
 └─ Bird
```

---

### Интерфейс

Когда нужно описать возможность:

```text
Bird умеет летать
Plane умеет летать

=> IFlyable
```

---

## Кратко

```text
Абстрактный класс:
- отношение "является"
- может содержать реализацию

Интерфейс:
- отношение "умеет"
- задаёт контракт
```
