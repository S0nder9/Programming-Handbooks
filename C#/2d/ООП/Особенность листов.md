## Особенность `List<T>`

`List<T>` — ссылочный тип.

Это означает, что при присваивании копируется не сам список, а ссылка на него.

---

## Проблема

Допустим, в классе есть список:

```csharp
private List<Product> _products = new List<Product>();
```

и метод:

```csharp
public List<Product> GetProducts()
{
    return _products;
}
```

Тогда внешний код может получить доступ к внутреннему списку:

```csharp
List<Product> products = cart.GetProducts();

products.RemoveAt(0);
```

---

### Что произойдёт?

Будет изменён не только `products`, но и список внутри `Cart`.

Потому что:

```csharp
products
```

и

```csharp
_products
```

ссылаются на один и тот же объект в памяти.

---

### Схема

```text
products ───┐
            ▼
        List<Product>
            ▲
_products ──┘
```

Удаление элемента через одну ссылку изменит данные для всех ссылок.

---

## Как защитить внутренний список?

Не возвращать сам список наружу.

Вместо этого можно предоставить методы доступа:

```csharp
public Product GetProductByIndex(int index)
{
    return _products[index];
}

public int GetProductsCount()
{
    return _products.Count;
}
```

---

Тогда внешний код сможет читать данные:

```csharp
for (int i = 0; i < cart.GetProductsCount(); i++)
{
    Console.WriteLine(cart.GetProductByIndex(i).Name);
}
```

но не сможет удалить элементы из `_products`.

---

## Другой способ — вернуть копию списка

```csharp
public List<Product> GetProducts()
{
    return new List<Product>(_products);
}
```

Теперь:

```csharp
List<Product> products = cart.GetProducts();
products.RemoveAt(0);
```

не повлияет на список внутри `Cart`.

---

## Кратко

```text
List<T> — ссылочный тип.

Если вернуть список напрямую:
→ внешний код сможет его изменить.

Чтобы защитить данные:
- возвращают копию списка;
- либо предоставляют методы доступа.
```
