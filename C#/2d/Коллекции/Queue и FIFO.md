## `Queue<T>` и принцип FIFO

`Queue<T>` — это коллекция, которая работает по принципу **FIFO** (*First In, First Out*).

Перевод:

```text
Первым пришёл — первым ушёл
```

Так работает обычная очередь:
- человек встал первым,
- человек обслуживается первым.

---

## Создание очереди

```csharp
Queue<string> patients = new Queue<string>();
```

Создаём очередь пациентов.

---

## Добавление в очередь

```csharp
patients.Enqueue("Василий");
patients.Enqueue("А");
patients.Enqueue("Р");
patients.Enqueue("Вл");
```

Метод:

```csharp
Enqueue()
```

добавляет элемент в конец очереди.

После добавления:

```text
Василий → А → Р → Вл
```

---

## Извлечение элемента

```csharp
Console.WriteLine(patients.Dequeue());
```

Метод:

```csharp
Dequeue()
```

- возвращает первый элемент;
- удаляет его из очереди.

Результат:

```text
Василий
```

Очередь теперь:

```text
А → Р → Вл
```

---

## Просмотр первого элемента

```csharp
Console.WriteLine(patients.Peek());
```

Метод:

```csharp
Peek()
```

возвращает первый элемент, но не удаляет его.

Результат:

```text
А
```

Очередь останется:

```text
А → Р → Вл
```

---

## Перебор очереди

```csharp
foreach (var patient in patients)
{
    Console.WriteLine(patient);
}
```

Вывод:

```text
А
Р
Вл
```

---

## Количество элементов

```csharp
patients.Count
```

Возвращает количество элементов в очереди.

---

## Очистка очереди

```csharp
patients.Clear();
```

Удаляет все элементы.

---

## Основные методы

```csharp
Enqueue()   // добавить в конец
Dequeue()   // взять первый и удалить
Peek()      // посмотреть первый
Clear()     // очистить очередь
Count       // количество элементов
```

---

## Где используется `Queue<T>` в реальных проектах?

`Queue<T>` применяется везде, где задачи должны обрабатываться **в порядке поступления**.

### Серверы и веб-приложения

Запросы пользователей могут помещаться в очередь и обрабатываться по одному или группами.

Например:
- отправка email после регистрации;
- генерация PDF-документов;
- обработка загруженных файлов.

---

### Игры

Очередь действий персонажа:

```text
Атака
↓
Перезарядка
↓
Бросок гранаты
```

Игрок нажимает несколько кнопок подряд, а игра выполняет действия по очереди.

---

### Обработка сообщений

В крупных системах сервисы обмениваются сообщениями через очереди.

Например:

```text
Пользователь оформил заказ
↓
Сообщение попадает в очередь
↓
Сервис оплаты обрабатывает заказ
↓
Сервис доставки получает информацию
```

На практике для этого часто используются системы вроде :contentReference[oaicite:0]{index=0} или :contentReference[oaicite:1]{index=1}.

---

### Планировщики задач

Фоновые задачи помещаются в очередь:

```text
Создать отчёт
↓
Сжать файл
↓
Отправить уведомление
```

Специальный процесс берёт задачи из очереди и выполняет их по порядку.

---

### Обработка файлов

Если одновременно загружено много файлов:

```text
file1.pdf
file2.pdf
file3.pdf
```

их можно добавить в очередь и обрабатывать по одному, чтобы не перегружать сервер.

---

### Главное правило

Используй `Queue<T>`, когда действует принцип:

```text
Кто пришёл первым — тот обработан первым.
```
---

## Схема FIFO

```text
Enqueue("Василий")
Enqueue("А")
Enqueue("Р")

Очередь:

Василий → А → Р

Dequeue()

А → Р
```

Первый добавленный элемент всегда выходит первым.
