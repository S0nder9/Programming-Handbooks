## `Stack<T>` и принцип LIFO

`Stack<T>` — это коллекция, которая работает по принципу **LIFO** (*Last In, First Out*).

Перевод:

```text
Последним вошёл — первым вышел
```

Представь стопку тарелок:

```text
[5] ← верхушка
[4]
[3]
[2]
[1]
```

Чтобы достать тарелку, нужно взять верхнюю.

---

## Создание стека

```csharp
Stack<int> numbers = new Stack<int>();
```

---

## Добавление элементов

```csharp
numbers.Push(1);
numbers.Push(2);
numbers.Push(3);
numbers.Push(4);
numbers.Push(5);
```

Метод:

```csharp
Push()
```

добавляет элемент на вершину стека.

После добавления:

```text
Верх
 ↓
[5]
[4]
[3]
[2]
[1]
```

---

## Просмотр верхнего элемента

```csharp
Console.WriteLine(numbers.Peek());
```

Метод:

```csharp
Peek()
```

возвращает верхний элемент, но не удаляет его.

Результат:

```text
5
```

Стек остаётся без изменений.

---

## Удаление верхнего элемента

```csharp
numbers.Pop();
```

Метод:

```csharp
Pop()
```

- возвращает верхний элемент;
- удаляет его из стека.

После удаления:

```text
Верх
 ↓
[4]
[3]
[2]
[1]
```

---

## Перебор стека

```csharp
foreach (var item in numbers)
{
    Console.WriteLine(item);
}
```

Вывод:

```text
4
3
2
1
```

Перебор идёт от вершины к основанию.

---

## Извлечение всех элементов

```csharp
while (numbers.Count > 0)
{
    Console.WriteLine(numbers.Pop());
}
```

Результат:

```text
4
3
2
1
```

После цикла стек станет пустым.

---

## Основные методы

```csharp
Push()   // добавить наверх
Pop()    // получить и удалить верхний элемент
Peek()   // посмотреть верхний элемент
Clear()  // очистить стек
Count    // количество элементов
```

## Где используется Stack в реальных проектах?

### История браузера

Когда нажимаешь кнопку "Назад":

```text
Страница C
Страница B
Страница A
```

Браузер возвращается к последней открытой странице.

---

### Отмена действий (Undo)

В редакторах:

```text
Напечатал текст
↓
Удалил слово
↓
Изменил цвет
```

При нажатии `Ctrl + Z` действия отменяются в обратном порядке.

---

### Вызов методов

Во время работы программы CLR использует **стек вызовов (Call Stack)**:

```text
Main()
 ↓
MethodA()
 ↓
MethodB()
```

После завершения `MethodB` управление возвращается в `MethodA`.

---

### Обход деревьев и графов

Стек часто используется в алгоритмах:

- DFS (Depth First Search);
- обход файловых директорий;
- анализ выражений и компиляция кода.

---

## Главное правило

Используй `Stack<T>`, когда действует принцип:

```text
Последний добавленный элемент
должен быть обработан первым.
```
