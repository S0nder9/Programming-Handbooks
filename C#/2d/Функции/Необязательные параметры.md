## Необязательные параметры

Необязательные параметры позволяют задать **значение по умолчанию**, которое будет использоваться, если аргумент не передан при вызове метода.

---

## Пример

```csharp
static string Color(
    string text,
    ConsoleColor color = ConsoleColor.Red,
    char symbol = '!')
{
    return text + symbol;
}
```

Здесь:

```csharp
ConsoleColor color = ConsoleColor.Red
```

и

```csharp
char symbol = '!'
```

— необязательные параметры.

---

## Вызов без необязательных параметров

```csharp
Console.WriteLine(Color("280"));
```

Результат:

```text
280!
```

Будут использованы значения по умолчанию:

```csharp
color = ConsoleColor.Red
symbol = '!'
```

---

## Передача одного параметра

```csharp
Console.WriteLine(Color("280", ConsoleColor.Green));
```

Результат:

```text
280!
```

Здесь:

```csharp
color = ConsoleColor.Green
symbol = '!'
```

---

## Именованные параметры

Можно передавать аргументы по имени:

```csharp
Console.WriteLine(Color("280", symbol: ')'));
```

Результат:

```text
280)
```

В этом случае:

```csharp
color = ConsoleColor.Red
symbol = ')'
```

Параметр `color` не был указан, поэтому используется значение по умолчанию.

---

## Зачем нужны именованные параметры?

Они позволяют передавать только нужные аргументы, не соблюдая порядок.

Например:

```csharp
Color(
    text: "Error",
    symbol: '?'
);
```

Код становится более читаемым.

---

## Правило

Необязательные параметры всегда должны идти в конце списка параметров:

✅ правильно

```csharp
void Test(string text, int count = 0)
```

❌ ошибка

```csharp
void Test(int count = 0, string text)
```
