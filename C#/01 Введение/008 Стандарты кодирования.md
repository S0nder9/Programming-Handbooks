# Стандарты кодирования

1. Имя переменной должно соответствовать ее назначению.
2. Имя функции должно соответствовать ее функциональности в коде.
3. Комментарии в коде
4. Форматирование и стиль кода
5. Структура проекта и организация кода
6. Git и код-ревью стандарты

---

## Обязательные в серьёзных проектах:

`SOLID` — обязательно знать и применять.
`Clean Code` — принципы Роберта Мартина.
`Domain-Driven Design` (базовые понятия) — особенно в микросервисах.
`CQRS` (часто спрашивают).
`Vertical Slice Architecture или Clean Architecture` (современный тренд).

## Лучшие практики .NET:

`Async/Await` — не блокируй поток (Task.Wait(), .Result — почти всегда плохо).
Используй `IAsyncEnumerable`<T> где нужно.
`Record` для DTO и Value Objects.
`Immutable` по возможности (init-only свойства, record).
`Nullability` — всегда включай <Nullable>enable</Nullable>.
`Pattern Matching` (C# 8–13) — используй активно.
`Expression-bodied` членов где уместно.
`Primary Constructors` (C# 12).
